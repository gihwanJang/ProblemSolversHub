# 풀이

### 생각의 흐름
해당 문제를 풀기 위해 저의 생각의 흐름은 다음과 같이 진행되었습니다.

1. 강의실 수, 예약 수를 입력받고, 예약 명단을 리스트로 입력받는다.
2. 예약 명단은 예약 수만큼 반복해서 2차원 리스트로 입력받는다.
3. check_request 함수를 통해 예약 확인 여부를 출력 받는다.

### 예약 확인 여부
예약에서 가장 중요한 것은 현재 예약하고 싶은 강의실에 이미 예약된 것이 있느냐입니다.

고려해야될 사항은 '이미 확약된 예약의 종료 시간'이 '지금 해야할 예약의 시작 시간'보다 같거나 빨랴아합니다.

때문에 강의실의 개수 만큼 '끝나는 시간(end_times)을 저장할 배열'을 만들고 현재 예약 시작시간과 비교해줍니다.

# 느낀점 및 수정해야될 사항

- YES/NO를 출력할 때 시간을 줄이기 위해 한번에 출력하면 좋을 것 같습니다.
